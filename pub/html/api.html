Detailed description of all the functions and the configuration you can do with SQL101.

Quiz Maker

Function

.createMCQuiz(problems, table, idName, options)

Parameters

Note: The text inside «» indicates the type of the variable.
  • problems «[Object]»
      • Object :
    • instruction «String» // the instruction to be displayed
    • question «String» // the question to be displayed
    • choices «[String]» // the available choices for the user to select from
    • answer «[String]» // the correct answer
  • table «Object»
      • Object :
    • attributes «[String]» // the attributes for the table (column names)
    • data «[Object]» // object keys should be from the attributes, the value is the corresponding value to that attribute
  • idName «String» // the identifier used to modify the DOM elements
  • options «[String]» // pass in the options here ("marks", "time")

CSS IDs

Note: The text inside «» should be replaced with the value of the parameters.
Note 2: The «type» can be either "MC" for multiple choice or "MI" for make instance.
  • answerDiv_«idName»
  • instructionsDiv_«idName»
  • questionDiv_«idName»
  • answeringForm_«idName»
  • «choice»«idName»
  • submitButton_«idName»
  • prevButton_«idName»
  • nextButton_«idName»
  • startButton_«idName»
  • timeElapsed_«idName»
  • feedbackDiv_«idName»
  • table«type»_«idName»
  • tableDiv«type»_«idName»

Algorithm Visualization

Function

.animationClosure(attributeSet, FDSet, speed, idName)

Parameters

Note: The text inside «» indicates the type of the variable.
  • attributeSet «String» // the set of attributes you want to calculate the closure for
  • FDSet «[String]» // an array of the FDs, in the form of "LHS->RHS"
  • speed «String» // the initial speed of the animations
  • idName «String» // the identifier used to modify the DOM elements

CSS IDs

Note: The text inside «» should be replaced with the value of the parameters.
  • inputDiv_«idName»
  • buttonsDiv_«idName»
  • speedUpButton_«idName»
  • speedDownButton_«idName»
  • nextStepButton_«idName»
  • algorithmDiv_«idName»

Parameters

Note: The text inside «» indicates the type of the variable.
  • table1 «Object»
      • Object :
    • attributes «[String]» // the attributes for the table (column names)
    • data «[Object]» // object keys should be from the attributes, the value is the corresponding value to that attribute
